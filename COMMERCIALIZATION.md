# Commercialization Boundary

## Current License

This repository is released under the Apache License 2.0. The current public workspace, documentation, scripts, and sample project code are intended to stay usable as open source assets.

## Why Apache-2.0

Apache-2.0 keeps the early project easy to adopt, fork, learn from, and reuse. It also provides an explicit patent grant and clear attribution terms, which are practical for a technical workspace and project templates.

## Future Commercial Options

When the Sirius ecosystem becomes more mature, commercial work should be separated instead of making this public root repository ambiguous. Reasonable options:

1. Keep this root workspace and public templates under Apache-2.0.
2. Put commercial products, hosted services, private integrations, or proprietary modules in separate repositories.
3. Use dual licensing only for clearly separated modules where the ownership and dependency boundary are clean.
4. Use commercial service agreements for deployment, integration, customization, support, and operations.

## What Should Stay Open

- Workspace operating rules
- Public documentation and templates
- Reusable audit scripts
- Generic project scaffolds
- Public examples that do not expose private infrastructure

## What May Become Commercial

- Hosted Sirius platform services
- Enterprise integrations
- Private deployment automation
- Customer-specific modules
- Branded UI kits or productized distributions
- Advanced workflow orchestration that depends on private infrastructure

## Brand and Trademark Boundary

The Apache-2.0 license covers source and documentation. It does not grant permission to use Sirius branding in a way that implies endorsement, official status, or ownership of the Sirius Coding project.
