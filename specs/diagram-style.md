# Diagram Style Spec

> 目标：让流程图具备信息设计能力，而不是只把文本机械渲染成图。

## 视觉方向

- 风格：ProcessOn / 企业流程图 / 干净信息图。
- 背景：纯白或接近白色，避免复杂纹理。
- 卡片：低饱和填充色、清晰描边、轻阴影、圆角不超过 8px。
- 箭头：正交线为主，避免斜线穿插；反馈回路走外围。
- 文字：中文优先清晰可读，不截断，不压线。

## 尺寸与间距

| 元素 | 规则 |
| --- | --- |
| 主链卡片 | 宽 360-400px，高 86-100px，居中排列 |
| 分支卡片 | 宽 160-180px，高 96-112px，横向等宽排列 |
| 主链间距 | 垂直间距 110-130px |
| 分支组 | 用虚线边框包裹，内部留白不少于 14px |
| 汇聚节点 | 位于分支组正下方并保持水平居中 |
| 反馈线 | 从末端节点走画布外围回到起点，不能穿过主体 |

## 语义颜色

| 角色 | 色彩规则 |
| --- | --- |
| `main` | 蓝、绿、橙、紫等低饱和卡片，保持主链节奏 |
| `hub` | 蓝色系，表示知识分流或控制中心 |
| `branch` | 分支之间可轻微区分颜色，但整体保持同一层级 |
| `merge` | 红色或粉色系，表示公开/安全/发布前护栏 |
| `end` | 蓝色或绿色系，表示公开输出和下一轮增强 |

## 版式规则

1. 主链必须一眼可读，优先垂直居中。
2. 分支必须横向展开、顶部对齐、等宽排列。
3. 汇聚节点必须回到主轴中心。
4. 箭头不穿过文字和卡片，不交叉。
5. 反馈回路必须走外围，不干扰主体阅读。

## 质量自检

渲染前后都要检查：

- 主链是否一眼可读。
- 分支是否对齐。
- 是否有箭头交叉。
- 是否存在布局拥挤。
- 反馈回路是否干扰主体。
